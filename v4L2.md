# 架构详解

​	V4L2子系统 采用分层架构,从上至下为: 用户空间APi, V4L2核心层,中间辅助层, 底层硬件驱动层

​	中间辅助层: DMA/VDMA驱动, ISP适配层, 总线层(spi/i2c/csi-2)等在音视频驱动使用过程中,所需要使用的.

## 核心组件

### 设备节点:`struct video_device`

保存V4L2设备节点的数据

​	驱动向用户空间暴露的核心对象, 对应了`/dev/videoX`的设备节点,负责注册字符设备, 处理文件操作

(`open, release, ioctl, read, write, mmap, poll`)

```c
//	关键成员
struct video_device{
    fops: 指向 v4l2_file_operations，定义文件操作。
    ioctl_ops: 指向 v4l2_ioctl_ops，定义具体的 V4L2 ioctl 回调。
    v4l2_dev: 指向父设备 v4l2_device。
    queue: 指向 vb2_queue，管理视频缓冲队列。
}

```



这个结构体需要设置这些参数:

- v4l2_dev: 设置 v4l2_device 作为父设备。

- name: 设置为唯一的描述性设备名。

- fops: 设置为已有的 v4l2_file_operations 结构体。

- ioctl_ops: v4l2_ioctl_ops 结构体.

- lock: 如果你要在驱动中实现所有的锁操作，则设为 NULL 。

#### video_device注册

```c
video_register_device(vdev, VFL_TYPE_VIDEO, -1);
//X通常传入-1,系统自动选择空闲编号
VFL_TYPE_VIDEO: 用于视频输入/输出设备的 videoX
VFL_TYPE_VBI: 用于垂直消隐数据的 vbiX (例如，隐藏式字幕，图文电视)
VFL_TYPE_RADIO: 用于广播调谐器的 radioX
```



### 视频缓冲队列: `videobuf2`

​	管理视频缓冲区, 负责内存分配, DMA映射, 缓冲区队列管理以及流控制

​	核心结构体: `struct vb_queue`, `struct vb2_buffer`

​	支持的内存模型: 

- `VB2_MEMORY_MMAP`: 内核分配内存, 映射到用户空间
- `VB2_MEMORY_USERPTR`: 用户空间分配内存, 传递指针给内核
- `VB2_MEMORY_DMABUF`: 基于`DMA_BUF`的内存共享机制, 用于不同设备的内存共享(零拷贝)

### 控制接口: `v4l2_ctrl_handler`

​	标准化的方式管理设备参数(对比度,亮度等参数)

功能如下: 

- 自动参数验证（范围、步进）。
- 生成控制菜单。
- 处理控制事件通知。
- 支持 32 位和 64 位值。

### 子设备: `struct v4l2_subdev`

子设备实例

​	视频设备,通常由多个ic组成, 使用`v4l2_subdev`来抽象这些组件

- **用途**：表示 Sensor、ISP、Video Decoder/Encoder 等。
- **通信** ：通过 `v4l2_subdev_ops` 定义内部操作，支持通知回调。
- **用户空间访问** ：可以创建 `/dev/v4l-subdevX` 节点供用户空间直接配置子设备。

控制子设备的统一接口,内部包含了需要实现的函数指针

```c
//初始化
v4l2_subdev_init()
//向v4l2_device注册subdev
v4l2_device_register_subdev()
//注册完成后,可以使用以下宏定义访问subdev的操作函数
v4l2_subdev_call(sd, core, g_std, &norm);//(等同于sd->ops->core->g_std(sd, &norm);)

```



### 媒体控制器: `Media Controller`

​	V4L2结合媒体控制器框架,以此来描述设备内部的复杂拓扑关系,(如多个输入,多个处理, 多个输出)

- **实体 (Entity)**：功能单元（如 Sensor, ISP）。
- **端口 (Pad)**：连接点（输入/输出）。
- **链接 (Link)**：连接两个 Pad 的通路。
- **应用** ：用户空间可以通过 `/dev/mediaX` 配置 pipeline 路由和格式。

### v4l2_device

保存设备实例的数据,一般会将其嵌入到一个更大的结构体

通过以下函数注册设备实例

```c
v4l2_device_register(struct device *dev, struct v4l2_device *v4l2_dev)
```

# V4L2子设备

## Subdev 注册

### 注册子设备

1. 注册同步子设备

   此方法用于子设备确定链接的情况下

   ```c
   v4l2_device_register_subdev(v4l2_dev,sd)
   ```

   此函数会将,`subdev->dev`指向`v4l2_device`,并且在`v4l2_device.mdev`(媒体设备)存在的情况下,自动注册到媒体设备

   子设备的注销

   ```c
   v4l2_device_unregister_subdev (sd)
   ```

   

2. 注册异步子设备

   此方法用于子设备链接时间未知的情况下(一般使用这种方法,isp和sensor都是异步注册)

   ```
   v4l2_async_register_subdev(v4l2_dev,sd)
   ```

   子设备的注销

   ```
   v4l2_async_unregister_subdev() 
   ```

3. 异步子设备通知器

   - 对于isp驱动来说,必须注册这个成员

   ```
   v4l2_async_nf_register() 调用执行的。要注销通知器，驱动程序必须调用 v4l2_async_nf_unregister()。在释放未注册通知器的内存之前，必须通过调用 v4l2_async_nf_cleanup() 来清理它。
   ```

   在注册之前需要

   ```
   在注册通知器之前，桥接器驱动程序必须做两件事：首先，必须使用 v4l2_async_nf_init() 初始化通知器。其次，桥接器驱动程序可以开始形成桥接器设备运行所需的异步连接描述符列表。v4l2_async_nf_add_fwnode()、v4l2_async_nf_add_fwnode_remote() 和 v4l2_async_nf_add_i2c()
   ```

   - 对于子设备来说,也可以注册通知器,当该子设备还需要其他子设备的时候

     ```
     v4l2_async_subdev_nf_init() 初始化通知器
     ```

4. 摄像头传感器驱动的异步子设备注册函数(5.1内核及以后)

   ```
   v4l2_async_register_subdev_sensor() 是传感器驱动程序注册自己的异步连接的辅助函数，但它也注册一个通知器，并进一步注册固件中找到的镜头和闪光设备的异步连接。子设备的通知器使用 v4l2_async_unregister_subdev() 注销并与异步子设备一起清理
   ```

## I2C子设备驱动(摄像头驱动)

使用`v4l2_subdev_init (sd, &ops)`初始化`v4l2_subdev`结构体

辅助函数位`v4l2-common.h`

- 添加从设备到`i2c`的方法是,将结构体`v4l2_subdev`内嵌到设备结构体中'

  ```c
  //示例设备结构体
  struct chipname_state {
          struct v4l2_subdev sd;
          ...  /* additional state fields */
  };
  ```

  通过以下函数初始化`subdev`, 

  ```c
  //自动填写subdev中所有字段,使得subdev和i2c_client相互指向
  v4l2_i2c_subdev_init(&state->sd, client, subdev_ops);
  ```

- 应该实现一个辅助**内联**函数, 该函数作用为,通过`subdev`,寻找设备实例结构体的地址

  ```c
  static inline struct chipname_state *to_state(struct v4l2_subdev *sd)
  {
          return container_of(sd, struct chipname_state, sd);
  }
  ```

- 从 `v4l2_subdev` 结构转到 `i2c_client` 结构	

  ```c
  struct i2c_client *client = v4l2_get_subdevdata(sd);
  ```

- 从 `i2c_client` 结构转到`v4l2_subdev` 结构

  ```c
  struct v4l2_subdev *sd = i2c_get_clientdata(client);
  ```

- `remove()`函数中需要调用

  ```c
   v4l2_device_unregister_subdev()(sd)
  ```


## 集中管理的子设备活动状态

​	子设备的状态通过`struct v4l2_mbus_framefmt`[]结构体数组表示, 每一pad引脚对应一个条目

```c
struct v4l2_mbus_framefmt {
    __u32 width;           // 宽度（像素）
    __u32 height;          // 高度（像素）
    __u32 code;            // 格式代码（如 MEDIA_BUS_FMT_SGRBG10_1X10）
    __u32 field;           // 场模式（逐行/隔行）
    __u32 colorspace;      // 色彩空间
    __u16 ycbcr_enc;       // YCbCr编码
    __u16 quantization;    // 量化范围
    __u16 xfer_func;       // 传输函数
    __u16 flags;           // 标志位
    __u16 reserved;
};
```



# 源码解析

## `v4l2_device`初始化

```c
// 关键函数：v4l2_device_register
int v4l2_device_register(struct device *dev, struct v4l2_device *v4l2_dev)
{
    if (v4l2_dev == NULL)
        return -EINVAL;

    INIT_LIST_HEAD(&v4l2_dev->subdevs); // 初始化子设备链表
    spin_lock_init(&v4l2_dev->lock);    // 初始化锁
    v4l2_prio_init(&v4l2_dev->prio);    // 初始化优先级管理
    kref_init(&v4l2_dev->ref);          // 初始化引用计数
    get_device(dev);
    v4l2_dev->dev = dev;
    
    // 设置名称
    if (!v4l2_dev->name[0])
        snprintf(v4l2_dev->name, sizeof(v4l2_dev->name), "%s %s",
            dev->driver->name, dev_name(dev));
            
    // 将 v4l2_device 设为 driver data
    if (!dev_get_drvdata(dev))
        dev_set_drvdata(dev, v4l2_dev);
        
    return 0;
}

```

## 结构体总结

```c
/*
	数据格式
*/
struct datafmt{
	u32 code;			//图像数据格式
	enum v4l2_colorspace;	//色彩空间格式
};

/*
	设备结构体
*/
struct xxx{
	struct v4l2_subdev;		//子设备, i2c/spi传感器设备
	const struct datafmt*;	//数据格式
	struct v4l2_rect;		//图像裁剪相关参数
	struct v4l2_clk*;		//时序结构体
	
	int total_width;		//总图像宽度 = rect.图像裁剪宽度 + 消隐宽度
	int total_height;		//总图像高度 = rect.图像裁剪高度 + 消隐高度
    
};

/*
	图像裁剪参数
*/
struct v4l2_rect{
    __s32   left;			//图像左侧起始点
	__s32   top;			//图像上方起始点
	__u32   width;			//图像宽度
	__u32   height;			//图像高度
};
/*
	图像帧格式
*/
struct v4l2_mbus_framefmt {
	__u32			width;
	__u32			height;
	__u32			code;
	__u32			field;
	__u32			colorspace;
...
};

/*
	摄像头裁剪能力
*/
struct v4l2_cropcap {
	__u32			type;	/* enum v4l2_buf_type ,缓冲区类型,是视屏输出还是输入*/
	struct v4l2_rect        bounds;		//裁剪区域的边界
	struct v4l2_rect        defrect;	//默认的裁剪矩形
	struct v4l2_fract       pixelaspect;	//内部分子和分母设置宽高比
};

/*
	媒体总线参数(CSI..)
*/
struct v4l2_mbus_config {
	enum v4l2_mbus_type type;		//媒体总线类型
	unsigned int flags;				//总线配置标志位,可以通过"|",多个一起使用
};
```
